# WoS、Zotero 文献整理与问卷完善设计

日期：2026-07-19

## 目标

以验证“引擎无关、科学语义完整、可验证的量子化学到 Blender 工作流”真实需求为第一目标，利用 Zotero 已有全文和 Web of Science 定向检索补强证据，完善 12–15 分钟的中文问卷。

## 输入与约束

- Zotero 父分类：`PySCF × ChemBlender 量子化学可视化调研（2026-07-19）`；内部 key 仅保存在已忽略的 `tmp/`。
- 当前条目：53 条；已附 PDF 37 条；缺 PDF 16 条。
- Web of Science 每日限额 50 次。本轮计划使用 6–10 次，硬上限 12 次；每次调用记录查询式、结果数和用途。
- 优先使用 Zotero 已有全文；WoS 用于发现缺失研究和核对书目信息，不为扩大数量而检索。
- 不使用 Sci-Hub 或其他绕过付费墙的来源。
- 用户隐私、学校访问方式、凭据、个人身份信息和含隐私的原始记录只能写入已被忽略的 `tmp/`；不得进入 Git 可跟踪文件、提交记录或公开调研产物。

## 证据工作流

1. 盘点 53 条条目的元数据、PDF 和全文可读状态。
2. 按问卷构念建立证据缺口：实际工作流、可视化行为、痛点与失败后果、信任与验证、概念价值、功能优先级、采用障碍。
3. 使用 WoS 定向检索缺口；按标题、摘要、文献类型、时间和主题相关性筛选。
4. 对已在 Zotero 的文献补充缺失 DOI、摘要、期刊、卷期页和作者；新增文献先查重再导入。
5. 将 WoS 新增且最终纳入的文献同步写入根目录 BibTeX，保证 Zotero、工作簿和 BibTeX 的标识符一致。
6. 将条目放入主题子分类，并把直接支撑问卷构念的条目交叉加入 `00｜优先精读`。

## 缺 PDF 优先级

对仍缺 PDF 的条目评分并输出下载清单：

- P0：直接决定问卷构念、题干或选项，且没有可替代全文证据；
- P1：用于解释关键结果或支持分层分析；
- P2：背景、软件功能或案例材料，可暂用摘要和元数据。

每条 P0/P1 文献说明其对应的问卷模块和需要从全文确认的内容。

如果 P0/P1 文献无法从现有 Zotero 附件或合法开放来源取得，生成精选下载清单并暂停最终结论和问卷定稿。目标保持进行中，直到用户通过学校权限补充全文并明确确认“精选文献已经补全”；不以摘要替代关键全文证据。

## 问卷结构

最终问卷控制在 12–15 分钟，PySCF 作为解释变量而非中心：

1. 受访者背景与过去 12 个月实际工作流；
2. 三维场可视化对象、工具和使用频率；
3. 痛点、错误风险和现有替代方式；
4. 元数据、跨引擎验证和科学可信度要求；
5. 引擎无关 Blender 工作流概念测试；
6. 功能优先级、采用意愿和部署方式；
7. PySCF 认知、使用与障碍；
8. 开放案例和后续访谈。

题目保留唯一变量名、题型、必答状态和可执行分支逻辑；删除重复、诱导或无法对应研究问题的题目。
