# 问卷选项减负与 ChemBlender 2.1 能力衔接设计

## 目标

在不增加 18 道题和现有角色覆盖面的前提下，减少受访者逐项阅读负担。问卷应优先收集会直接影响量子化学场可视化开发路线的信息；背景、协作和采用条件只保留足够分层分析的粒度。

## 设计依据

### 受访负担

当前 17 道封闭题共有 193 个可见选项。多数背景题与辅助分析题超过 6 项，容易让受访者在到达场类型、文件格式和功能优先级等核心问题前产生疲劳。

### ChemBlender 2.1 已有能力

对 `ChemBlender_2_x` 仓库 `snapshot/20260707-current-state` 分支的首个独有提交 `78c2d8d` 进行只读源码检查。该提交声明版本为 2.1.0，已有功能主要包括：

- 分子和晶体结构导入、编辑与 Geometry Nodes 表示；
- 球棍模型、晶胞、超胞、截面、配位多面体、热椭球和对称操作；
- 快速渲染、结构文件导出和基础分子力场操作；
- CIF、SDF、MOL、XYZ、PDB、POSCAR/VASP 等结构数据入口，以及 CIF、MOL、SDF、XYZ、VASP 输出。

源码中未发现 Gaussian Cube、体数据、等值面或 OpenVDB 导入实现。`CH_Atomic Orbital` 是 Geometry Nodes 预设，不能视为已经支持计算所得轨道场。因此问卷不重复罗列已有结构建模功能，而是重点确认量子化学场与这些结构能力衔接时的真实需求。

## 选项压缩策略

采用语义合并，不按现有顺序简单截断。只有当多个选项对应同一个开发动作或同一种后续分析用途时才合并。

- Q08 场类型：约 10 项，保留分子轨道、密度/势、激发态派生场、周期场和时变场等开发差异。
- Q10 文件格式：约 12 项，继续细问；这是 ChemBlender 2.1 的明确能力缺口，并决定解析器优先级。
- Q11 可视化痛点：约 9 项，围绕导入、科学语义、性能、协作复现和误读风险合并。
- Q13 功能优先级：约 10 项，保留场导入与结构对齐、等值面/体渲染/截面、相位与状态、周期体系、大网格、多帧、证据记录、分享和自动化。
- 其余封闭题：原则上压缩到 5–6 项；保留必要的“不确定”“没有需求”或“其他”出口。

预计总选项由 193 项降至约 118 项，减少约 39%。具体措辞以语义完整和互斥性优先，不为满足精确总数而强行合并。

## ChemBlender 能力在问卷中的表达

Q13 不把已有功能拆成多个候选项，只保留一个衔接需求：

> 量子化学场与现有分子/晶体结构、周期超胞和截面功能联动

其余候选项描述 2.1 尚未覆盖或需要加强的开发能力。问卷不使用“已有”“计划支持”等宣传性表述，也不要求受访者了解 ChemBlender。

Q10 保持为条件题，仅向亲自运行计算、参与参数设置或准备工作流的人显示。问题询问实际处理或能够获取的量子化学场格式，不把普通结构格式混入其中。

## 开场说明

问卷开场使用两段融合文字，不再单独解释单选、多选和选择上限：

> 本调查旨在了解量子化学/电子结构计算结果从产生、交付到理解、核验和三维场可视化的实际流程，并据此确定 ChemBlender 的后续更新方向。ChemBlender 目前主要支持分子与晶体结构的导入、编辑和展示；后续可能扩展量子化学场与波函数文件导入、场数据与分子/晶体结构的正确对齐、等值面与体渲染、正负相位和状态标识、周期体系与大网格处理、多帧动画、计算与可视化参数记录，以及结果分享和协作交付等能力。本问卷将重点了解这些方向在真实研究工作中的使用场景、优先级和采用障碍。
>
> 无论您是亲自运行计算，还是与计算研究者合作、委托代算或主要使用计算结果，都可以根据实际经历作答。如果某项技术细节通常由合作者或服务方处理，可以选择“不确定”或最接近实际情况的选项，不必猜测程序、参数或文件格式。

## RDKit 安装方式偏好题

在“可接受的交付方式”之后、“首要采用障碍”之前新增 Q16。原 Q16–Q18 顺延为 Q17–Q19。

题干简要说明两种方式的取舍：内置固定 wheel 支持离线安装且较少改动 Blender Python 环境，但会增大安装包；运行时 pip 安装可以减小安装包，但依赖网络和权限，也可能安装失败或改动 Blender Python 环境。

Q16 为必答单选题，变量名为 `rdkit_install_preference`，选项为：

- Extension 内置固定版本 wheel
- 运行时通过 pip 联网安装
- 没有明确偏好，只要安装后能够正常使用

## 选择限制

- 一般多选题最多选择 2–3 项。
- Q13 功能优先级最多选择 4 项。
- Q08 和 Q10 不设置过低上限，避免多体系、多软件用户被迫遗漏真实需求。
- 单选题维持单选，不增加排序题或矩阵题。

## 不做的内容

- 除 RDKit 安装偏好题外，不再增加题目、角色分支或 ChemBlender 使用经历题。
- 不把问卷改成 ChemBlender 2.1 满意度调查。
- 不逐项验证 Geometry Nodes 中每个通用建模节点的需求。
- 不把源码中未验证的功能描述成现有能力。

## 验证标准

- 题号保持 Q01–Q19 连续，19 个变量名唯一。
- 18 道封闭题共有 121 个选项；新增 Q16 为 3 项单选题。
- 大多数封闭题为 5–6 个选项，只有 Q08、Q10、Q11 和 Q13 明显超过 6 项。
- 所有选项继续使用缩进多行 Markdown 列表。
- Q03 到 Q10 的显示逻辑仍然存在，Q10 仍为条件多选题。
- Q13 同时覆盖 ChemBlender 2.1 能力衔接和量子化学场可视化新增能力。
- README、研究综述和问卷说明中的题数、预计时长及链接保持一致。
- Git 可跟踪文件的隐私扫描无命中，`tmp/` 不进入提交。
