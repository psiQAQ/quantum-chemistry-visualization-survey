<h1>Survey 定向阅读问题</h1><p><strong>关联模块：</strong> OpenQP、激发态与开放互操作</p><p>阅读时请直接在每个"回答"后补充结论，并记录对应页码、图表或原文位置。</p><h2>阅读问题</h2><h3>问题 1</h3><p>MRSF-TDDFT 的结果最需要哪些态、轨道和跃迁可视化？</p><p><strong>回答：</strong> MRSF-TDDFT（Mixed-Reference Spin-Flip TDDFT, Sec. 2.2）是 OpenQP 的标志性方法，产生的可视化需求包括：(1) 旋翻激发态（spin-flip excited states）——常规 TDDFT 无法处理的 open-shell 单重态-三重态混合激发，需要展示自旋翻转前后的自旋密度变化；(2) 跃迁轨道对（natural transition orbitals, NTOs）——MRSF-TDDFT 的跃迁密度矩阵经奇异值分解后产生的主要粒子-空穴对轨道可视化（Sec. 2.2）；(3) DTCAM 系列交换-相关泛函（VAEE, XI, XIV, AEE, VEE, Sec. 2.3）产生的特异性轨道能量和空间分布差异；(4) 双激发（double excitation）特征——MRSF-TDDFT 能描述含有双激发成分的激发态，其跃迁密度的多元可视化需求超越传统 TDDFT；(5) 激发态能量和振子强度——吸收光谱模拟（Figure 4-5, 7-10）需要谱线归属和各激发态的跃迁密度可视化；(6) 锥形交叉区——MRSF-TDDFT 的解析梯度支持 conical intersection 搜索（PyOQP, Sec. 2.4），需要双态之间的轨道和密度差可视化。</p><h3>问题 2</h3><p>自主模块与第三方互联如何体现，适配器能力边界是否可机器读取？</p><p><strong>回答：</strong> OpenQP 采用模块化架构（Sec. 2.1），以"open-source ecosystem"模式（如 PySCF, Psi4 传统，Sec. 1）实现：(1) 自主模块——OpenQP 将 HF、DFT、TDDFT、SF-TDDFT、MRSF-TDDFT 各方法实现为独立模块（oqp.set_basis, oqp.scf, oqp.tddft 等），模块间通过结构化数据（"mol"类型）连接（Sec. 2.1, Figure 1）；(2) 第三方互联——C API 接口（Sec. 2.1）实现数据交换，允许第三方软件调用特定模块；PyOQP Python 封装（Sec. 2.4）提供几何优化、锥形交叉搜索和非绝热耦合计算；(3) 适配器边界——当前接口能力限制：(a) 不支持自动数据格式转换（如直接输出 Molden/Cube）；(b) 未提供通用的文件 I/O 标准，数据交换依赖 C API 结构体内的数据指针；(c) 尚未与 ASE 等外部环境直接集成；(4) 机器可读能力——结构化数据类型的定义使得模块签名（输入/输出类型）可由程序检查，但跨程序适配器的能力描述尚未标准化（无 schema 或 OpenAPI 式的规范）。</p><h3>问题 3</h3><p>Python 包装、高性能并行和开源生态分别解决哪些采用障碍？</p><p><strong>回答：</strong> OpenQP 的策略覆盖三个不同的障碍层：(1) Python 包装（PyOQP, Sec. 2.4）——解决可用性障碍：自动处理几何优化和锥形交叉搜索，降低输入编写门槛；利用 Python 生态（NumPy, SciPy）进行后处理，降低数据分析门槛；提供与 PySCF 类似的用户体验（Sec. 1），降低学习曲线；(2) 高性能并行（BLAS/LAPACK, MPI via base library, Sec. 2.1）——解决性能障碍：提供与 Fortran/C++ 相当的计算性能，使得 MRSF-TDDFT 适用于中等分子体系（约 50-200 原子），支持并行执行以加速大规模计算；(3) 开源生态（Sec. 1）——解决持续性障碍：开源发布确保代码可持续访问和社区贡献，模块化架构允许外部开发者贡献新的理论方法模块（Sec. 6），基于 BSD/MIT 兼容许可（未明确指定，但文中强调"open-source ecosystem"）降低商业/学术采用的许可顾虑。可视化方面的障碍仍需独立解决——OpenQP 目前未提供内置可视化输出。</p><h2>阅读后回填</h2><ul><li>对问卷题目的影响：可增加关于"模块化架构对可视化工具集成的影响"和"PyOQP 与外部可视化工具互操作"的题目</li><li>可转化为访谈追问：OpenQP 用户是否需要 MRSF-TDDFT 特有的自旋-翻转跃迁可视化功能</li><li>关键证据页码/图表：Sec. 1 (开源生态), Sec. 2.1 是否有计划支持通用输出格式（Cube/Molden/HDF5）？</li></ul><h2>综合摘要</h2><p>本文（Mironov et al., J. Chem. Theory Comput. 20, 9464-9477, 2024）介绍了 OpenQP 开源量子化学平台，以模块化架构（Sec. 2.1）解决可持续性和互操作性挑战。核心科学创新是 MRSF-TDDFT（Sec. 2.2）及其 DTCAM 系列泛函（Sec. 2.3），能描述含双激发特征的旋翻激发态。技术架构上采用自主模块间结构化数据类型传参、C API 支持第三方互联、PyOQP Python 封装（Sec. 2.4）提供工作流自动化。可视化需求涵盖 NTO 粒子-空穴对、自旋翻转产生的自旋密度差、锥形交叉双态密度差和吸收光谱归属。模块化 API 设计为外部可视化工具集成提供了可能，但原生输出格式和标准化适配器定义仍有待发展。</p>
