# 项目协作规则

## 研究目标

本项目调研量子化学三维场可视化需求，重点验证一个引擎无关、科学语义完整、可验证的量子化学计算到 Blender 工作流是否具有真实需求。PySCF 采用与认可度是解释变量，不是问卷的唯一中心。

## 隐私边界

- 用户身份、学校或机构信息、学校访问方式、账号、令牌、Cookie、本机私密路径、未脱敏访谈记录和其他隐私内容只能写入 `tmp/`。
- `tmp/` 必须保持在 `.gitignore` 中，不得强制加入 Git。
- 根目录、`docs/`、提交记录、公开表格和图片不得出现上述隐私信息。
- 公开前对所有 Git 可跟踪文件执行隐私扫描；不确定的信息按隐私内容处理。

## 文献检索与证据

- Web of Science 每日限额为 50 次；本轮计划使用 6–10 次，硬上限 12 次。
- 每次 WoS 调用必须在 `tmp/wos_search_log.md` 记录查询式、用途、返回数和纳入数。
- 优先使用 Zotero 已有全文；WoS 只补证据缺口，不为扩大条目数而检索。
- 不使用 Sci-Hub 或其他绕过付费墙的来源。
- 摘要和元数据可用于初筛，但不能替代直接决定问卷构念、题干或选项的关键全文证据。

## Zotero 与公开资料同步

- Zotero 父分类 `IMD6DTWK` 是本轮文献状态源；允许一个条目进入多个主题子分类，但不得复制条目。
- 新条目先按 DOI 查重；无 DOI 时使用标题相似度查重。
- 最终纳入的新条目必须同步到根目录 BibTeX 和工作簿，保持 DOI、标题、作者和年份一致。
- 更新工作簿后必须重新渲染并检查预览图；更新 BibTeX 后必须检查重复 key、重复 DOI 和语法。

## 全文门槛

- `P0`：直接决定问卷构念、题干或选项，且没有可替代全文证据。
- `P1`：用于解释关键结果或支持主要分层分析。
- `P2`：背景、软件功能或案例材料，可暂用摘要和元数据。
- 若仍缺 P0/P1 全文，生成精选下载清单并暂停最终结论和问卷定稿。
- 只有用户明确确认“精选文献已经补全”后，才能继续最终定稿并把目标标记为完成。

## 验证要求

- 回读 Zotero 父分类、子分类、条目数和附件数。
- 核对 Zotero、BibTeX、工作簿中的纳入文献与 DOI 集。
- 检查问卷题号连续、变量名唯一、分支目标存在、选项无明显诱导，预计时长为 12–15 分钟。
- 不用未验证的数量、检索结果或附件状态声明完成。
